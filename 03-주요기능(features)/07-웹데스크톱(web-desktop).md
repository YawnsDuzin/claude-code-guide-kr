# Claude Code Web & Desktop 사용법

터미널 CLI 외에도 Web과 Desktop 앱에서 Claude Code를 사용할 수 있습니다.

---

## 접근 방식 비교

| 구분 | Terminal CLI | VS Code 확장 | Desktop 앱 | Web |
|------|:-----------:|:----------:|:---------:|:---:|
| **Free** | ✅ | ✅ | ❌ | ❌ |
| **Pro** | ✅ | ✅ | ✅ | ✅ |
| **Max** | ✅ | ✅ | ✅ | ✅ |
| **설치 필요** | npm | VS Code 마켓 | 다운로드 | 없음 |
| **오프라인** | ❌ | ❌ | ❌ | ❌ |

---

## Web (claude.ai/code)

### 시작하기

1. [claude.ai/code](https://claude.ai/code)에 접속
2. Claude Pro/Max 계정으로 로그인
3. GitHub 저장소를 연결하거나 새 프로젝트 시작

### 주요 특징

- **브라우저에서 바로 사용**: 설치 없이 어디서든 접근 가능
- **GitHub 연동**: 저장소를 직접 연결하여 코드 편집
- **실시간 미리보기**: 웹 앱 변경사항을 즉시 확인
- **공유 가능**: 세션 URL을 통해 결과물 공유

### 사용 흐름

```
1. claude.ai/code 접속 및 로그인
2. "New Project" 또는 GitHub 저장소 연결
3. 자연어로 작업 요청
4. Claude가 코드 편집, 터미널 명령 실행
5. 변경사항 확인 후 커밋/푸시
```

### Web에서 할 수 있는 것

```
"이 프로젝트에 로그인 기능을 추가해줘"
"README.md를 작성해줘"
"테스트를 실행하고 실패하는 것을 수정해줘"
"npm run build 결과를 확인해줘"
```

### 제한사항

- 로컬 파일 시스템에 직접 접근 불가
- 복잡한 로컬 개발 환경(Docker, DB 등) 사용 제한
- 장시간 실행되는 프로세스는 타임아웃 가능

## Desktop 앱

### 설치

| OS | 다운로드 |
|----|---------|
| **macOS** | [claude.ai](https://claude.ai)에서 다운로드 |
| **Windows** | [claude.ai](https://claude.ai)에서 다운로드 |

### 주요 특징

- **로컬 파일 접근**: 로컬 프로젝트 폴더를 직접 열어 작업
- **네이티브 성능**: 브라우저보다 빠른 반응 속도
- **시스템 통합**: 파일 드래그 앤 드롭, 알림 등
- **멀티 프로젝트**: 여러 프로젝트를 탭으로 관리

### 사용 흐름

```
1. Desktop 앱 실행
2. 프로젝트 폴더 열기 (File > Open Folder)
3. 자연어로 작업 요청
4. Claude가 로컬 파일 편집, 터미널 명령 실행
5. 변경사항 확인
```

### Desktop vs Terminal CLI

| 기능 | Desktop | Terminal CLI |
|------|---------|-------------|
| 파일 트리 보기 | ✅ 시각적 | ❌ 텍스트 |
| diff 미리보기 | ✅ 시각적 | ✅ 텍스트 |
| 이미지 첨부 | ✅ 드래그앤드롭 | ✅ 파일 경로 |
| 멀티 세션 | ✅ 탭 | ✅ 터미널 여러 개 |
| 자동화/스크립트 | ❌ | ✅ 파이프라인 |
| CLAUDE.md | ✅ 자동 로드 | ✅ 자동 로드 |
| MCP 서버 | ✅ | ✅ |

## VS Code 확장

### 설치

1. VS Code에서 확장 마켓플레이스 열기 (`Ctrl+Shift+X`)
2. "Claude Code" 검색
3. 설치 후 VS Code 재시작

### 사용 방법

```
# VS Code에서 Claude Code 패널 열기
Ctrl+Shift+P → "Claude Code" 입력

# 또는 사이드바의 Claude Code 아이콘 클릭
```

### 주요 기능

- **에디터 통합**: 현재 열린 파일을 자동으로 컨텍스트에 포함
- **인라인 diff**: 변경사항을 에디터에서 바로 확인/적용
- **터미널 통합**: VS Code 내장 터미널에서 명령 실행
- **선택 영역 전송**: 코드 선택 후 Claude에게 바로 질문

### 실전 사용 예시

```
# 선택한 코드에 대해 질문
1. 코드 영역 선택
2. 우클릭 → "Ask Claude Code"
3. "이 함수를 최적화해줘"

# 현재 파일 전체에 대해 요청
"이 파일의 버그를 찾아줘"
"테스트 코드를 작성해줘"
```

## 어떤 것을 선택해야 할까?

| 상황 | 추천 |
|------|------|
| 터미널에 익숙한 개발자 | **Terminal CLI** |
| VS Code를 주로 사용 | **VS Code 확장** |
| 빠르게 코드 수정이 필요할 때 | **Web** |
| 로컬 프로젝트를 GUI로 작업 | **Desktop** |
| CI/CD 파이프라인에서 사용 | **Terminal CLI** |
| 팀원에게 결과 공유 | **Web** |
| 무료로 사용하고 싶을 때 | **Terminal CLI** 또는 **VS Code** |

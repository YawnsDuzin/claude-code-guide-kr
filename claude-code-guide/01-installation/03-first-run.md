# 첫 실행 및 초기 설정

Claude Code 설치를 완료했다면 이제 처음 실행하고, 인증을 설정하고, 첫 프로젝트에서 사용해 봅시다. 이 문서를 따라하면 10분 안에 Claude Code를 활용할 준비가 됩니다.

---

## 처음 실행하기

터미널을 열고 `claude` 명령어를 입력합니다:

```bash
claude
```

처음 실행하면 다음과 같은 과정이 순서대로 진행됩니다:

1. **이용 약관 동의** 확인
2. **인증** (API 키 또는 OAuth 로그인)
3. **권한 모드** 선택
4. 대화형 세션 시작

---

## 인증 설정

Claude Code는 두 가지 인증 방식을 지원합니다. 첫 실행 시 원하는 방식을 선택할 수 있습니다.

### 방식 1: Anthropic API 키

API 키를 직접 입력하는 방식입니다. 사용량에 따라 과금됩니다.

**사전 설정 (환경 변수에 미리 저장해두기)**:

```bash
# .bashrc 또는 .zshrc에 추가
export ANTHROPIC_API_KEY=sk-ant-api03-xxxxxxxxxxxxx

# 설정 적용
source ~/.bashrc   # 또는 source ~/.zshrc
```

환경 변수가 설정되어 있으면 `claude` 실행 시 자동으로 해당 API 키를 사용합니다.

**실행 시 직접 입력**:

환경 변수를 설정하지 않은 경우, 첫 실행 시 인증 방법을 묻는 화면이 나타납니다. "Anthropic API Key"를 선택하고 키를 입력하면 됩니다.

### 방식 2: OAuth 로그인 (Claude 구독 사용자)

Claude Max, Team, Enterprise 구독이 있다면 OAuth 로그인을 사용할 수 있습니다.

1. `claude` 실행 후 인증 방법에서 "Claude.ai account (OAuth)"를 선택합니다
2. 브라우저가 자동으로 열립니다
3. Claude 계정으로 로그인합니다
4. 권한 승인을 완료합니다
5. 터미널로 돌아오면 인증이 완료되어 있습니다

> **WSL 사용자 참고**: WSL에서 OAuth를 사용하면 Windows 기본 브라우저가 자동으로 열립니다. 브라우저가 열리지 않으면 터미널에 표시되는 URL을 직접 브라우저에 복사해서 접속하세요.

### 인증 상태 확인

```bash
# 현재 인증 상태 확인
claude config
```

## 권한 모드 선택

인증이 완료되면 **권한 모드(Permission Mode)**를 선택합니다. 권한 모드는 Claude가 파일을 수정하거나 명령어를 실행할 때 사용자의 승인을 요구하는 수준을 결정합니다.

### 세 가지 권한 모드

| 모드 | 설명 | 권장 대상 |
|------|------|-----------|
| **기본 모드 (Default)** | 파일 수정과 셸 명령어 실행 시 사용자 승인 요청. 읽기 작업은 자동 허용 | 대부분의 사용자 (초보자에게 추천) |
| **완화 모드 (Relaxed)** | 안전한 작업(파일 쓰기 등)은 자동 허용. 파괴적일 수 있는 명령어만 승인 요청 | Claude Code에 익숙한 사용자 |
| **엄격 모드 (Strict)** | 모든 작업에 대해 사용자 승인 요청 | 보안이 중요한 환경 |

### 권한 모드 동작 예시

**기본 모드에서의 흐름**:

```
You: login 함수에 이메일 형식 검증을 추가해줘

Claude: login.js 파일을 수정하겠습니다.

  [파일 수정 내용 미리보기]

  이 변경사항을 적용할까요? (y/n/e)
  y - 수정 적용
  n - 수정 거부
  e - 에디터에서 수정 후 적용
```

### 권한 모드 변경

나중에 권한 모드를 변경할 수 있습니다:

```bash
# 세션 시작 시 권한 모드 지정
claude --permission-mode default
claude --permission-mode relaxed
claude --permission-mode strict
```

> **초보자 팁**: 처음에는 **기본 모드(Default)**를 사용하세요. Claude가 어떤 작업을 수행하는지 하나하나 확인하면서 동작 방식을 학습할 수 있습니다. 익숙해지면 완화 모드로 전환하여 작업 속도를 높이세요.

## 첫 프로젝트에서 사용해 보기

### 프로젝트 디렉토리로 이동

Claude Code는 **현재 디렉토리를 작업 컨텍스트**로 사용합니다. 반드시 프로젝트 루트 디렉토리에서 실행하세요.

```bash
# 프로젝트 디렉토리로 이동
cd ~/my-project

# Claude Code 실행
claude
```

### 프로젝트 구조 파악하기 (추천 첫 질문)

처음 사용할 때는 프로젝트 구조를 물어보는 것이 좋습니다. Claude가 프로젝트를 분석하고 이해하게 됩니다.

```
> 이 프로젝트의 구조와 주요 기능을 설명해줘
```

Claude가 다음과 같은 내용을 분석하여 알려줍니다:
- 디렉토리 구조
- 사용된 프레임워크와 기술 스택
- 주요 진입점(entry point)
- 설정 파일들의 역할
- 의존성(dependencies)

### 코드 이해하기

특정 파일이나 함수에 대해 질문할 수 있습니다:

```
> src/auth/login.js 파일에서 인증 로직이 어떻게 동작하는지 설명해줘

> handleSubmit 함수가 호출되는 곳들을 모두 찾아줘

> 이 프로젝트에서 데이터베이스 연결은 어떻게 설정되어 있어?
```

### 간단한 수정 요청하기

프로젝트를 이해했다면, 간단한 수정부터 시작해 보세요:

```
> README.md에 프로젝트 설치 방법 섹션을 추가해줘

> utils.js에 이메일 형식을 검증하는 함수를 만들어줘

> 이 함수에 에러 처리를 추가해줘
```

## 기본 명령어

Claude Code 세션 내에서 사용할 수 있는 기본 명령어입니다. 모두 `/`로 시작합니다.

### 필수 명령어

| 명령어 | 설명 | 사용 예시 |
|--------|------|-----------|
| `/help` | 사용 가능한 명령어 목록과 도움말 표시 | `/help` |
| `/clear` | 현재 대화 기록 초기화. 컨텍스트가 너무 커졌을 때 유용 | `/clear` |
| `/exit` | Claude Code 세션 종료. `Ctrl+C`를 두 번 눌러도 됨 | `/exit` |

### 자주 사용하는 명령어

| 명령어 | 설명 |
|--------|------|
| `/compact` | 현재 대화를 요약하여 컨텍스트 공간 확보 |
| `/status` | 현재 세션 상태 확인 (모델, 토큰 사용량 등) |
| `/model` | 사용할 AI 모델 변경 |
| `/permissions` | 권한 설정 확인 및 변경 |
| `/config` | Claude Code 설정 확인 및 변경 |

### 명령어 사용 예시

```
# 대화 시작
> 이 프로젝트를 설명해줘

# ... 여러 대화 후 컨텍스트가 커지면
> /compact

# 대화 내역이 요약되어 컨텍스트가 줄어듭니다
# 새로운 주제로 전환할 때 유용합니다

> /clear
# 대화 기록이 완전히 초기화됩니다
# 완전히 새로운 작업을 시작할 때 사용합니다
```

## CLAUDE.md 설정하기

`CLAUDE.md`는 프로젝트 루트에 놓는 특별한 파일입니다. Claude Code가 세션 시작 시 이 파일을 자동으로 읽어서 프로젝트의 규칙과 맥락을 파악합니다.

### CLAUDE.md를 만들어야 하는 이유

- Claude에게 프로젝트의 **코딩 컨벤션**을 알려줄 수 있습니다
- **자주 사용하는 명령어**(빌드, 테스트, 린팅 등)를 기록해 둘 수 있습니다
- 프로젝트의 **아키텍처와 중요한 결정사항**을 설명할 수 있습니다
- 매번 같은 설명을 반복하지 않아도 됩니다

### CLAUDE.md 기본 템플릿

프로젝트 루트에 `CLAUDE.md` 파일을 생성하세요:

```markdown
# 프로젝트 개요

이 프로젝트는 [프로젝트 설명]입니다.

## 기술 스택

- 프론트엔드: React 18, TypeScript
- 백엔드: Node.js, Express
- 데이터베이스: PostgreSQL
- 테스트: Jest

## 개발 명령어

- 개발 서버 시작: `npm run dev`
- 테스트 실행: `npm test`
- 빌드: `npm run build`
- 린팅: `npm run lint`

## 코딩 컨벤션

- TypeScript strict 모드 사용
- 함수형 컴포넌트와 React Hooks 사용
- 변수명은 camelCase, 컴포넌트명은 PascalCase
- 모든 함수에 JSDoc 주석 작성

## 프로젝트 구조

src/
├── components/    UI 컴포넌트
├── hooks/         커스텀 React Hooks
├── services/      API 통신 로직
├── utils/         유틸리티 함수
└── types/         TypeScript 타입 정의

## 주의사항

- .env 파일은 절대 커밋하지 말 것
- API 키는 환경 변수를 통해 관리
- 데이터베이스 마이그레이션 후 반드시 `npm run seed` 실행
```

### Claude에게 CLAUDE.md 생성을 요청하기

직접 작성하기 어렵다면, Claude에게 요청할 수도 있습니다:

```
> 이 프로젝트를 분석하고 CLAUDE.md 파일을 만들어줘
```

Claude가 프로젝트 구조, package.json, 설정 파일 등을 분석하여 적절한 CLAUDE.md를 자동으로 생성합니다.

### CLAUDE.md 계층 구조

CLAUDE.md는 여러 위치에 놓을 수 있으며, 범위가 다릅니다:

| 위치 | 범위 | 설명 |
|------|------|------|
| `프로젝트루트/CLAUDE.md` | 프로젝트 전체 | 프로젝트 공통 규칙, 팀원과 공유됨 |
| `프로젝트루트/.claude/CLAUDE.md` | 프로젝트 전체 | 개인 설정용 (`.gitignore`에 추가 가능) |
| `하위디렉토리/CLAUDE.md` | 해당 디렉토리 | 특정 모듈/디렉토리에만 적용되는 규칙 |
| `~/CLAUDE.md` | 모든 프로젝트 | 전역 설정 (개인 코딩 스타일 등) |

## 유용한 시작 팁

### 1. 구체적으로 요청하세요

```
# 좋은 예
> src/utils/validation.js에 이메일 형식을 검증하는 validateEmail 함수를 추가해줘.
> 정규식을 사용하고, 유효하지 않으면 에러 메시지를 반환해줘.

# 아쉬운 예
> 이메일 검증 만들어줘
```

### 2. 한 번에 하나의 작업을 요청하세요

처음에는 하나의 작업을 요청하고, 결과를 확인한 후 다음 작업을 요청하는 것이 좋습니다.

```
# 좋은 흐름
> 먼저 User 모델의 구조를 보여줘
> (확인 후) 여기에 email 필드의 유효성 검증을 추가해줘
> (확인 후) 관련 테스트도 작성해줘
```

### 3. 변경사항을 꼭 확인하세요

Claude가 파일을 수정하면 diff(변경사항 비교)를 보여줍니다. 적용하기 전에 꼭 확인하세요.

### 4. Git과 함께 사용하세요

Claude Code로 작업하기 전에 현재 상태를 커밋해 두면, 문제가 생겼을 때 쉽게 되돌릴 수 있습니다.

```bash
# 작업 전 현재 상태 저장
git add -A && git commit -m "작업 전 상태 저장"

# Claude Code로 작업
claude

# 결과가 마음에 들면 커밋
git add -A && git commit -m "Claude Code로 이메일 검증 기능 추가"
```

## 문제 해결

### "인증 실패" 또는 "API 키가 유효하지 않음"

```bash
# API 키가 올바르게 설정되었는지 확인
echo $ANTHROPIC_API_KEY

# API 키 재설정
export ANTHROPIC_API_KEY=sk-ant-api03-xxxxxxxxxxxxx
```

### "컨텍스트 윈도우 초과" 메시지

대화가 길어져서 컨텍스트가 가득 찬 상태입니다:

```
> /compact
# 또는
> /clear
```

### Claude가 잘못된 파일을 수정하려 할 때

수정 확인 프롬프트에서 `n`을 입력하여 거부하고, 더 구체적으로 요청하세요:

```
(y/n/e) n

> src/auth/login.js 파일만 수정해줘. 다른 파일은 건드리지 마.
```

### 응답이 매우 느린 경우

- 네트워크 연결 상태를 확인하세요
- 매우 큰 파일을 분석 중일 수 있습니다. 잠시 기다려 주세요
- `/compact`로 대화를 요약하면 응답 속도가 개선될 수 있습니다

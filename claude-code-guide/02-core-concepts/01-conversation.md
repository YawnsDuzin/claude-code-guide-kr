# 대화(Conversation) 기본 개념

Claude Code는 터미널 기반의 AI 대화 도구입니다. 이 문서에서는 Claude Code와 효과적으로 대화하는 방법을 알아봅니다.

---

## Claude Code 대화의 작동 방식

Claude Code는 일반적인 채팅 앱과 달리 **터미널(CLI) 환경**에서 동작합니다. 터미널을 열고 `claude` 명령어를 입력하면 대화형 세션이 시작됩니다.

```bash
# Claude Code 대화 시작
claude
```

실행하면 프롬프트(`>`)가 나타나며, 자연어로 질문하거나 작업을 요청할 수 있습니다.

```
> 이 프로젝트의 구조를 설명해줘
```

Claude는 질문에 답하면서 필요한 경우 파일을 읽고, 코드를 수정하고, 터미널 명령어를 실행하는 등의 **도구(Tool)**를 자동으로 사용합니다. 이 과정에서 사용자에게 권한을 요청할 수도 있습니다.

### 기본 흐름

1. 사용자가 자연어로 요청을 입력
2. Claude가 요청을 분석하고 필요한 도구를 선택
3. 도구 실행 전 권한 확인 (설정에 따라)
4. 도구 실행 및 결과 확인
5. 결과를 바탕으로 응답 생성

---

## 단일 턴(Single-turn) vs 다중 턴(Multi-turn) 대화

### 단일 턴 대화

한 번의 질문과 답변으로 끝나는 대화입니다. 주로 `-p` (print) 플래그를 사용합니다.

```bash
# 단일 질문 후 바로 결과 출력
claude -p "package.json에서 의존성 목록을 알려줘"
```

단일 턴은 다음과 같은 경우에 유용합니다:

- 간단한 질문에 빠르게 답을 얻고 싶을 때
- 스크립트나 자동화 파이프라인에서 Claude를 사용할 때
- CI/CD 환경에서 코드 리뷰를 자동화할 때

### 다중 턴 대화

여러 번의 질문과 답변을 주고받는 대화입니다. 대화형 모드(`claude`)에서 자연스럽게 이루어집니다.

```
> src 폴더에 어떤 파일들이 있어?

Claude: src 폴더에는 다음 파일들이 있습니다...
  - index.ts
  - utils.ts
  - config.ts

> utils.ts 파일의 내용을 보여줘

Claude: utils.ts 파일의 내용은 다음과 같습니다...

> 거기에 날짜 포맷 함수를 추가해줘

Claude: formatDate 함수를 utils.ts에 추가했습니다...
```

다중 턴 대화의 장점:

- **맥락 유지**: 이전 대화 내용을 기억하므로 "거기", "그 파일" 같은 지시대명사 사용 가능
- **점진적 작업**: 복잡한 작업을 단계별로 진행 가능
- **피드백 반영**: Claude의 결과를 확인하고 수정 요청 가능

## 컨텍스트(Context)를 효과적으로 사용하기

Claude Code는 현재 작업 디렉터리를 기준으로 프로젝트를 이해합니다. 효과적인 컨텍스트 제공을 위해 다음 사항을 기억하세요.

### 작업 디렉터리 설정

```bash
# 프로젝트 루트에서 Claude를 시작하는 것이 좋습니다
cd ~/my-project
claude
```

### 구체적인 파일 언급

```
# 덜 효과적
> 설정 파일 수정해줘

# 더 효과적
> src/config/database.ts 파일에서 포트 번호를 5432로 변경해줘
```

### 충분한 배경 정보 제공

```
# 덜 효과적
> 버그 고쳐줘

# 더 효과적
> users API 엔드포인트에서 GET /users/:id 요청 시 404 에러가 발생해.
> userId가 문자열인데 숫자로 비교하고 있어서 그런 것 같아.
> src/routes/users.ts를 확인해줘.
```

## 파일과 코드를 Claude에 전달하는 방법

### 방법 1: 대화 중 파일 경로 언급

가장 간단한 방법입니다. Claude가 자동으로 파일을 읽어 분석합니다.

```
> src/utils/helper.ts 파일을 리팩토링해줘
```

### 방법 2: 파이프(Pipe) 입력 사용

터미널의 파이프 기능을 활용해 파일 내용이나 명령어 결과를 직접 전달할 수 있습니다.

```bash
# 파일 내용을 Claude에 전달
cat src/app.py | claude "이 코드를 설명해줘"

# 에러 로그를 Claude에 전달
cat error.log | claude "이 에러의 원인을 분석해줘"

# 명령어 실행 결과를 전달
git diff | claude "이 변경사항을 요약해줘"

# 테스트 실패 결과를 분석
npm test 2>&1 | claude "실패한 테스트의 원인을 분석하고 수정 방법을 알려줘"

# 여러 파일을 한 번에 전달
cat src/models/*.ts | claude "이 모델들의 관계를 설명해줘"
```

파이프 입력은 자동으로 비대화형(non-interactive) 모드로 동작합니다. 즉, 한 번의 응답 후 종료됩니다.

파이프와 `-p` 플래그를 결합하면 결과를 파일로 저장할 수도 있습니다:

```bash
cat README.md | claude -p "이 문서를 한국어로 번역해줘" > README_ko.md
```

### 방법 3: 대화 중 코드 블록 직접 입력

대화 중에 코드를 직접 붙여넣을 수도 있습니다.

```
> 다음 코드에서 문제점을 찾아줘:
>
> function add(a, b) {
>   return a - b;
> }
```

## 비대화형 모드: `claude -p`

`-p` (또는 `--print`) 플래그를 사용하면 비대화형(non-interactive) 모드로 실행됩니다. 결과를 출력하고 즉시 종료합니다.

```bash
# 기본 사용법
claude -p "현재 디렉터리의 파일 목록을 알려줘"

# 출력 형식 지정
claude -p "README.md 내용을 요약해줘" --output-format json

# 스크립트에서 활용
SUMMARY=$(claude -p "package.json의 프로젝트 설명을 한 줄로 요약해줘")
echo "프로젝트 요약: $SUMMARY"
```

### 자동화 활용 예시

```bash
# Git 커밋 메시지 자동 생성
git diff --staged | claude -p "이 변경사항에 대한 커밋 메시지를 영어로 작성해줘"

# 코드 리뷰 자동화
git diff main..feature-branch | claude -p "이 PR의 코드 리뷰를 해줘. 버그 가능성, 성능 이슈, 개선점을 확인해줘"

# 문서 생성
cat src/api/*.ts | claude -p "이 API 엔드포인트들의 문서를 마크다운으로 작성해줘"
```

### 대화형 모드와의 차이점

| 특성 | 대화형 (`claude`) | 비대화형 (`claude -p`) |
|------|-------------------|----------------------|
| 다중 턴 | 가능 | 불가능 (단일 턴만) |
| 권한 요청 | 사용자에게 확인 | 자동 거부 (안전 모드) |
| 세션 유지 | 유지됨 | 없음 |
| 사용 목적 | 대화형 개발 | 스크립트, 자동화 |
| 파이프 출력 | 지원 안 함 | 파일 리다이렉트 가능 |

## 대화 이력과 `/clear` 명령어

### 대화 이력 관리

Claude Code는 대화형 세션 중 모든 대화 내용을 컨텍스트 윈도우에 유지합니다. 이전 대화를 기반으로 후속 질문에 답할 수 있는 이유입니다.

```
> users 테이블의 스키마를 알려줘

Claude: users 테이블의 스키마는...

> 거기에 email 컬럼을 추가하는 마이그레이션을 만들어줘

Claude: (이전 대화에서 users 테이블 정보를 이미 알고 있으므로 바로 작업 가능)
```

### `/clear` - 대화 초기화

대화가 길어지면 컨텍스트 윈도우가 가득 차거나, 이전 대화의 맥락이 오히려 방해가 될 수 있습니다. 이때 `/clear`를 사용합니다.

```
> /clear
```

`/clear` 사용이 필요한 경우:

- **주제 변경**: 완전히 다른 작업을 시작할 때
- **컨텍스트 정리**: 불필요한 대화 기록이 쌓여 응답 품질이 떨어질 때
- **비용 절감**: 큰 컨텍스트는 더 많은 토큰을 소비하므로 비용이 증가
- **오류 누적 해소**: Claude가 잘못된 가정을 계속 유지할 때

### 대화 재개(Resume)

이전 세션으로 돌아가고 싶다면 `--resume` 또는 `--continue` 플래그를 사용합니다.

```bash
# 가장 최근 대화 이어하기
claude --continue
# 또는 축약형
claude -c

# 특정 세션 ID로 이어하기
claude --resume <session-id>

# 세션 목록에서 선택하여 재개
claude --resume
```

## 유용한 슬래시 명령어 모음

대화 중 사용할 수 있는 슬래시 명령어들입니다.

| 명령어 | 설명 |
|--------|------|
| `/clear` | 대화 기록 초기화 |
| `/compact` | 대화 내용 압축 (컨텍스트 절약) |
| `/help` | 도움말 표시 |
| `/cost` | 현재 세션의 토큰 사용량과 비용 확인 |
| `/doctor` | Claude Code 설치 상태 점검 |

## 효과적인 대화를 위한 팁

### 1. 명확하고 구체적으로 요청하기

```
# 모호한 요청
> 코드 개선해줘

# 명확한 요청
> src/utils/date.ts의 formatDate 함수에서:
> 1. TypeScript 타입을 더 엄격하게 지정해줘
> 2. edge case (null, undefined 입력) 처리를 추가해줘
> 3. JSDoc 주석을 추가해줘
```

### 2. 큰 작업은 단계별로 나누기

```
# 한 번에 너무 많은 것을 요청
> 전체 프로젝트를 React에서 Vue로 마이그레이션해줘

# 단계별로 나누어 요청
> 먼저 프로젝트 구조를 분석하고 React 컴포넌트 목록을 알려줘
> (결과 확인 후)
> Header 컴포넌트를 Vue로 변환해줘
> (결과 확인 후)
> 잘 됐어. 이제 Sidebar 컴포넌트를 변환해줘
```

### 3. 의도를 함께 설명하기

```
# 의도 없이
> 이 함수에 캐싱을 추가해줘

# 의도와 함께
> 이 함수가 DB를 너무 자주 호출해서 응답이 느려.
> 결과를 5분간 메모리에 캐싱해서 성능을 개선하고 싶어.
```

### 4. 제약 조건 명시하기

```
> Express.js 미들웨어를 작성해줘.
> 조건:
> - TypeScript로 작성
> - 외부 라이브러리 사용 금지
> - 에러 로깅은 winston 사용
> - 기존 src/middleware/ 디렉터리의 스타일을 따를 것
```

### 5. 결과 확인 후 피드백 주기

Claude가 코드를 수정한 후에는 반드시 결과를 확인하고 피드백을 제공하세요.

```
> 잘 됐는데, 에러 메시지를 한국어로 바꿔줘
> 그리고 로그 레벨을 warn에서 error로 변경해줘
```

### 6. 예상 결과 설명하기

```
> 이 함수를 리팩토링해줘.
> 입력: {"name": "Kim", "age": 25}
> 기대 출력: "Kim (25세)"
```

### 7. CLAUDE.md 활용하기

프로젝트 루트에 `CLAUDE.md` 파일을 만들어 프로젝트 관련 정보를 제공하면, 매번 설명할 필요 없이 Claude가 프로젝트를 잘 이해할 수 있습니다.

```markdown
# CLAUDE.md 예시

## 프로젝트 개요
이 프로젝트는 Next.js 기반 전자상거래 플랫폼입니다.

## 기술 스택
- Next.js 14 (App Router)
- TypeScript
- Prisma ORM
- PostgreSQL

## 코드 컨벤션
- 컴포넌트는 PascalCase, 유틸 함수는 camelCase
- 테스트 파일은 *.test.ts 형식
- 커밋 메시지는 Conventional Commits 규칙 따름

## 자주 사용하는 명령어
- npm run dev: 개발 서버 시작
- npm test: 테스트 실행
- npx prisma migrate dev: DB 마이그레이션
```

### 8. 실행 취소와 안전망

Claude가 원치 않는 변경을 했다면:

```bash
# Git으로 변경사항 되돌리기
git checkout -- <파일경로>

# 또는 전체 되돌리기
git stash
```

항상 **Git으로 버전 관리되는 프로젝트**에서 Claude를 사용하는 것이 안전합니다.

## 요약

| 기능 | 사용법 | 용도 |
|------|--------|------|
| 대화형 모드 | `claude` | 다중 턴 대화, 복잡한 작업 |
| 비대화형 모드 | `claude -p "질문"` | 단일 질문, 스크립트 자동화 |
| 파이프 입력 | `cat file \| claude "질문"` | 파일/결과 분석 |
| 대화 초기화 | `/clear` | 컨텍스트 정리, 주제 변경 |
| 대화 압축 | `/compact` | 컨텍스트 공간 확보 |
| 대화 재개 | `claude --continue` | 이전 세션 이어하기 |

효과적인 대화의 핵심은 **구체적이고 명확한 요청**, **단계별 접근**, 그리고 **적절한 컨텍스트 관리**입니다.

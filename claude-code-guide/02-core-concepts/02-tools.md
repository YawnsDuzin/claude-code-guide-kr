# 내장 도구(Built-in Tools)

Claude Code는 단순한 텍스트 대화를 넘어 **실제 작업을 수행**할 수 있습니다. 이것을 가능하게 하는 것이 바로 내장 도구(Built-in Tools)입니다. Claude는 사용자의 요청을 분석하여 적절한 도구를 자동으로 선택하고 실행합니다.

---

## 도구 개요

| 도구 | 기능 | 주요 용도 |
|------|------|-----------|
| **Read** | 파일 읽기 | 코드 분석, 파일 내용 확인 |
| **Edit** | 파일 수정 | 코드 수정, 버그 수정 |
| **Write** | 파일 생성 | 새 파일 작성, 전체 덮어쓰기 |
| **Bash** | 터미널 명령 실행 | 빌드, 테스트, Git 등 |
| **Glob** | 파일 패턴 검색 | 파일 찾기, 구조 파악 |
| **Grep** | 파일 내용 검색 | 코드 검색, 패턴 찾기 |
| **Agent** | 하위 에이전트 실행 | 복잡한 조사, 병렬 작업 |
| **TodoWrite** | 작업 추적 | 체크리스트 관리 |
| **WebFetch** | 웹 페이지 가져오기 | URL 내용 읽기 |
| **WebSearch** | 웹 검색 | 최신 정보 검색 |
| **NotebookEdit** | Jupyter 노트북 편집 | 노트북 셀 추가/수정 |

---

## Read: 파일 읽기

`Read` 도구는 로컬 파일 시스템에서 파일을 읽어옵니다. 코드 분석, 설정 확인, 문서 검토 등에 사용됩니다.

### 주요 특징

- 절대 경로를 사용하여 파일을 읽음
- 기본적으로 파일의 처음 2000줄을 읽음
- 큰 파일은 `offset`과 `limit`으로 범위 지정 가능
- 이미지 파일(PNG, JPG 등)도 읽을 수 있음 (멀티모달)
- PDF 파일도 지원 (대용량 PDF는 페이지 범위 지정 필요)
- Jupyter 노트북(.ipynb) 파일도 지원

### 실제 사용 예시

```
> src/config.ts 파일을 보여줘
```

Claude는 자동으로 `Read` 도구를 사용하여 파일 내용을 읽고 보여줍니다.

```
> package.json에서 dependencies 부분만 보여줘
```

특정 부분만 필요한 경우에도 자연어로 요청하면 Claude가 적절히 처리합니다.

```
> error-screenshot.png 스크린샷을 확인해줘
```

이미지 파일도 Read 도구로 읽어서 시각적으로 분석할 수 있습니다.

## Edit: 파일 수정

`Edit` 도구는 기존 파일의 특정 부분을 **정확하게 교체(replace)**합니다. 파일 전체를 다시 쓰는 것이 아니라, 변경이 필요한 부분만 수정합니다.

### 주요 특징

- `old_string`(기존 텍스트)을 `new_string`(새 텍스트)으로 교체
- 교체 대상 문자열이 파일 내에서 유일해야 함 (중복 시 에러)
- `replace_all` 옵션으로 모든 일치 항목을 한번에 교체 가능
- 들여쓰기(indentation)를 정확히 유지

### 실제 사용 예시

```
> src/api/users.ts에서 포트 번호를 3000에서 8080으로 변경해줘
```

Claude는 내부적으로 다음과 같이 Edit 도구를 호출합니다:

```
Edit: src/api/users.ts
  old_string: "const PORT = 3000"
  new_string: "const PORT = 8080"
```

```
> utils.ts에서 모든 console.log를 logger.info로 바꿔줘
```

`replace_all: true` 옵션을 사용하여 파일 내 모든 일치 항목을 교체합니다.

### Edit vs Write의 차이

| 상황 | 사용 도구 |
|------|-----------|
| 파일의 일부분만 수정 | **Edit** (권장) |
| 완전히 새로운 파일 생성 | **Write** |
| 파일 전체를 재작성 | **Write** |
| 변수명을 일괄 변경 | **Edit** (`replace_all`) |

## Write: 파일 생성

`Write` 도구는 새로운 파일을 만들거나 기존 파일을 완전히 덮어씁니다.

### 주요 특징

- 새 파일을 지정한 경로에 생성
- 기존 파일이 있으면 전체 내용을 덮어씀
- 파일 생성 전 Read로 기존 내용 확인 필요 (기존 파일인 경우)

### 실제 사용 예시

```
> src/utils/validator.ts 파일을 새로 만들어줘.
> 이메일과 전화번호 검증 함수를 포함해줘.
```

Claude는 Write 도구를 사용하여 새 파일을 생성합니다.

```
> .gitignore 파일을 Node.js 프로젝트에 맞게 새로 작성해줘
```

기존 파일을 완전히 새로운 내용으로 교체할 때도 Write를 사용합니다.

## Bash: 터미널 명령 실행

`Bash` 도구는 시스템의 터미널에서 명령어를 실행합니다. 빌드, 테스트, Git 작업, 패키지 설치 등 다양한 작업에 사용됩니다.

### 주요 특징

- 일반적인 터미널 명령어를 실행
- 작업 디렉터리는 호출 사이에 유지되지 않음 (절대 경로 사용 권장)
- 기본 타임아웃: 2분 (최대 10분까지 설정 가능)
- 백그라운드 실행 지원
- 보안상 민감한 명령어는 사용자 승인 필요

### 실제 사용 예시

```
> 프로젝트 의존성을 설치해줘
```

Claude가 실행하는 명령어:
```bash
npm install
```

```
> 테스트를 실행하고 결과를 알려줘
```

```bash
npm test
```

```
> 현재 브랜치의 최근 커밋 5개를 보여줘
```

```bash
git log --oneline -5
```

```
> 8080 포트를 사용 중인 프로세스를 찾아줘
```

```bash
lsof -i :8080
```

### Bash 도구 사용 시 주의사항

- `find`, `grep`, `cat` 등의 명령어 대신 전용 도구(Glob, Grep, Read)를 사용하는 것이 더 효율적
- 대화형 명령어(`vim`, `nano`, `git rebase -i` 등)는 지원하지 않음
- 파괴적 명령어(`rm -rf`, `git reset --hard` 등)는 신중하게 사용

## Glob: 파일 패턴 검색

`Glob` 도구는 파일 이름 패턴을 사용하여 파일을 빠르게 찾습니다. 프로젝트 구조를 파악하거나 특정 유형의 파일을 찾을 때 유용합니다.

### 주요 특징

- 글로브 패턴(예: `**/*.ts`) 지원
- 수정 시간 기준으로 정렬된 결과 반환
- 대규모 코드베이스에서도 빠르게 동작

### 글로브 패턴 문법

| 패턴 | 설명 | 예시 |
|------|------|------|
| `*` | 모든 문자 (디렉터리 구분자 제외) | `*.ts` - 현재 폴더의 .ts 파일 |
| `**` | 모든 디렉터리 (재귀) | `**/*.ts` - 모든 하위 폴더의 .ts 파일 |
| `?` | 단일 문자 | `file?.ts` - file1.ts, fileA.ts 등 |
| `{a,b}` | 여러 패턴 중 하나 | `*.{ts,tsx}` - .ts 또는 .tsx 파일 |

### 실제 사용 예시

```
> 프로젝트에서 모든 TypeScript 파일을 찾아줘
```

Claude가 내부적으로 실행:
```
Glob: pattern="**/*.ts"
```

```
> src/components 폴더에서 테스트 파일을 찾아줘
```

```
Glob: pattern="**/*.test.{ts,tsx}", path="src/components"
```

```
> 설정 관련 파일들을 모두 찾아줘
```

```
Glob: pattern="**/*config*"
```

## Grep: 파일 내용 검색

`Grep` 도구는 파일 내용에서 특정 텍스트나 패턴을 검색합니다. ripgrep 기반으로 매우 빠르게 동작합니다.

### 주요 특징

- 정규 표현식 지원
- 대소문자 구분/무시 옵션
- 파일 유형별 필터링
- 검색 결과 주변 컨텍스트(전후 줄) 표시
- 세 가지 출력 모드: `files_with_matches`, `content`, `count`

### 실제 사용 예시

```
> "TODO"가 포함된 모든 파일을 찾아줘
```

```
Grep: pattern="TODO", output_mode="files_with_matches"
```

```
> fetchUser 함수가 어디에서 정의되어 있는지 찾아줘
```

```
Grep: pattern="function fetchUser|const fetchUser|fetchUser\s*=", output_mode="content"
```

```
> TypeScript 파일에서 any 타입을 사용하는 곳을 모두 찾아줘
```

```
Grep: pattern=": any[^a-zA-Z]", type="ts", output_mode="content"
```

```
> 환경 변수 DATABASE_URL이 어디에서 사용되는지 찾아줘
```

```
Grep: pattern="DATABASE_URL", output_mode="content", -C=2
```

`-C=2` 옵션은 일치하는 줄 전후 2줄을 함께 보여줍니다.

## Agent: 하위 에이전트 실행

`Agent` 도구는 **별도의 하위 에이전트(subagent)**를 생성하여 독립적인 작업을 수행하게 합니다. 복잡한 조사나 여러 파일에 걸친 분석이 필요할 때 유용합니다.

### 주요 특징

- 메인 대화의 컨텍스트와 별도로 작동
- 복잡한 질문에 대해 깊이 있는 조사 수행
- 여러 파일을 탐색하며 정보를 종합

### 실제 사용 예시

```
> 이 프로젝트의 인증 시스템이 어떻게 구현되어 있는지 전체적으로 조사해줘
```

Claude는 Agent 도구를 사용하여 하위 에이전트를 생성합니다. 하위 에이전트는 여러 파일을 탐색하고, 코드를 분석하고, 결과를 종합하여 보고합니다.

```
> 이 프로젝트에서 사용되지 않는 의존성(unused dependency)을 찾아줘
```

하위 에이전트가 package.json의 의존성 목록과 실제 import 문을 비교 분석합니다.

## TodoWrite: 작업 추적

`TodoWrite` 도구는 복잡한 작업을 수행할 때 **체크리스트를 생성하고 관리**합니다. 작업의 진행 상황을 추적하는 데 사용됩니다.

### 주요 특징

- 작업 목록(체크리스트) 생성
- 각 항목의 완료 상태 추적
- 우선순위 지정 가능

### 실제 사용 예시

```
> 이 프로젝트의 보안 취약점을 점검해줘
```

Claude가 내부적으로 체크리스트를 생성합니다:

```
TodoWrite:
  - [ ] 의존성 취약점 스캔
  - [ ] 환경 변수 노출 확인
  - [ ] SQL 인젝션 가능성 점검
  - [ ] XSS 취약점 확인
  - [ ] 인증/인가 로직 점검
```

각 항목을 순서대로 수행하며 완료 표시를 합니다.

## Claude가 도구를 선택하는 방식

Claude는 사용자의 요청을 분석하여 **가장 적절한 도구를 자동으로 선택**합니다. 사용자가 도구를 명시적으로 지정할 필요는 없습니다.

### 도구 선택 로직

| 사용자 요청 | Claude가 선택하는 도구 |
|-------------|------------------------|
| "이 파일 보여줘" | **Read** |
| "함수 이름을 바꿔줘" | **Edit** |
| "새 파일을 만들어줘" | **Write** |
| "테스트를 실행해줘" | **Bash** |
| "모든 .tsx 파일을 찾아줘" | **Glob** |
| "이 함수가 어디서 호출되는지 찾아줘" | **Grep** |
| "전체 아키텍처를 분석해줘" | **Agent** (하위 에이전트) |
| "이 URL 내용을 확인해줘" | **WebFetch** |

### 도구 조합 사용

복잡한 요청에는 여러 도구가 순차적으로 사용됩니다.

**예시**: "users 모델에 phone 필드를 추가하고 관련 테스트를 업데이트해줘"

1. **Grep**: `users` 모델 파일 위치 검색
2. **Read**: 모델 파일 내용 확인
3. **Edit**: phone 필드 추가
4. **Grep**: 관련 테스트 파일 검색
5. **Read**: 테스트 파일 내용 확인
6. **Edit**: 테스트 코드 업데이트
7. **Bash**: `npm test` 실행하여 테스트 통과 확인

### 도구 사용 권한

일부 도구는 실행 전 사용자의 승인이 필요합니다. 권한 설정에 대한 자세한 내용은 [권한 모드와 보안](./permissions.md) 문서를 참고하세요.

| 도구 | 읽기 전용 | 승인 필요 (기본 모드) |
|------|-----------|----------------------|
| Read | O | X |
| Glob | O | X |
| Grep | O | X |
| Edit | X | O |
| Write | X | O |
| Bash | X | O (명령어에 따라) |

## 요약

- Claude Code의 내장 도구는 파일 읽기/쓰기, 검색, 터미널 명령 실행, 웹 접근 등을 지원합니다.
- 사용자는 자연어로 요청만 하면 Claude가 적절한 도구를 **자동으로 선택**합니다.
- 복잡한 작업에는 여러 도구가 **조합**되어 사용됩니다.
- 파일 수정이나 명령어 실행 같은 작업은 **사용자 승인**이 필요할 수 있습니다.
- 각 도구의 특성을 이해하면 더 효과적인 요청을 작성할 수 있습니다.

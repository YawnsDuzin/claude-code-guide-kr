# 토큰 사용량 및 비용 관리

> Claude Code의 비용 구조를 이해하고, 효율적으로 사용하는 방법을 알아봅니다.

---

## 1. 비용 구조 이해

### 토큰이란?

토큰(Token)은 AI 모델이 텍스트를 처리하는 기본 단위입니다. 한국어는 영어보다 더 많은 토큰을 사용합니다.

- 영어: 1단어 ≈ 1~2 토큰
- 한국어: 1글자 ≈ 2~3 토큰
- 코드: 1줄 ≈ 10~30 토큰

### 입력 vs 출력 토큰

| 구분 | 설명 | 비용 |
|------|------|------|
| **입력 토큰** | 프롬프트, 파일 내용, 컨텍스트 | 상대적으로 저렴 |
| **출력 토큰** | Claude의 응답, 생성된 코드 | 입력 대비 3~5배 비싼 편 |

### 모델별 비용 비교

| 모델 | 입력 (1M 토큰) | 출력 (1M 토큰) | 상대 비용 |
|------|---------------|---------------|----------|
| Haiku | 가장 저렴 | 가장 저렴 | 1x |
| Sonnet | 중간 | 중간 | ~3x |
| Opus | 가장 비쌈 | 가장 비쌈 | ~10x |

> **참고:** 정확한 가격은 [Anthropic 가격 페이지](https://www.anthropic.com/pricing)에서 확인하세요.

## 2. 사용량 모니터링

### /cost 명령어

현재 세션의 토큰 사용량을 확인합니다:

```
/cost
```

**출력 예시:**
```
Session cost: $0.42
  Input tokens:  125,000
  Output tokens:  15,000
  Cache reads:    80,000
  Cache writes:   45,000
```

### 실시간 모니터링 팁

- 대규모 작업 전후에 `/cost` 확인
- 예상보다 비용이 높으면 프롬프트 전략 재검토
- 일일/주간 사용량 추적 습관 만들기

## 3. 비용 절감 전략

### 전략 1: 모델 적절히 선택

```bash
# 간단한 작업은 Haiku로
claude --model haiku "이 함수의 리턴 타입을 알려줘"

# 일반 코딩은 Sonnet으로
claude --model sonnet "API 엔드포인트를 구현해줘"

# 복잡한 설계만 Opus로
claude --model opus "이 시스템의 아키텍처를 재설계해줘"
```

### 전략 2: /compact 적극 활용

컨텍스트가 커질수록 매 요청의 입력 토큰이 증가합니다:

```
# 20턴마다 또는 주제 변경 시
/compact
```

**효과:** 컨텍스트 크기를 50~70% 줄여 이후 요청의 입력 비용 절감

### 전략 3: 집중된 프롬프트 작성

```
# 비용 높음: 모호하고 넓은 요청
"이 프로젝트를 전체적으로 분석하고 개선점을 모두 알려줘"

# 비용 낮음: 구체적이고 좁은 요청
"src/utils/date.ts의 formatDate 함수에서
타임존 처리 버그를 수정해줘"
```

### 전략 4: 세션 분리

```bash
# 하나의 긴 세션 (비용 높음)
# 컨텍스트가 계속 누적되어 매 요청마다 전체를 전송

# 여러 짧은 세션 (비용 낮음)
claude -p "모델 설계해줘"      # 세션 1
claude -p "API 구현해줘"       # 세션 2 (깨끗한 컨텍스트)
claude -p "테스트 작성해줘"     # 세션 3
```

### 전략 5: CLAUDE.md로 반복 제거

매번 설명하는 것보다 CLAUDE.md에 한 번 작성하면 캐시 효과로 비용이 줄어듭니다.

### 전략 6: 파일 전체 읽기 줄이기

```
# 비용 높음: "이 파일 전체를 읽어줘" (5000줄 파일)
# 비용 낮음: "이 파일의 120~150줄을 읽어줘"
```

## 4. 사용 계획별 비교

### Claude Max (구독제)

| 항목 | 내용 |
|------|------|
| 가격 | 월 $100 (Max 5), $200 (Max 20) |
| 특징 | 정액제, 사용량 상한 있음 |
| 장점 | 비용 예측 가능, 초과 과금 없음 |
| 단점 | 한도 초과 시 대기 필요 |
| 적합 | 개인 개발자, 일상적 사용 |

### API 키 (종량제)

| 항목 | 내용 |
|------|------|
| 가격 | 사용한 토큰만큼 과금 |
| 특징 | 사용량 제한 없음 |
| 장점 | 유연한 사용, 대량 작업 가능 |
| 단점 | 비용 예측 어려움, 관리 필요 |
| 적합 | 팀/기업, 무거운 작업 |

### 선택 가이드

```
일일 사용 < 2시간 → Max 5 ($100/월)
일일 사용 2~6시간 → Max 20 ($200/월)
일일 사용 > 6시간 또는 CI/CD → API 키
팀 5명 이상 → Team/Enterprise 플랜
```

## 5. 작업별 예상 비용

| 작업 유형 | 예상 토큰 | 대략적 비용 (Sonnet) |
|----------|----------|---------------------|
| 단순 질문/수정 | 5K~15K | < $0.01 |
| 함수 구현 | 15K~50K | $0.01~0.05 |
| 파일 생성 | 30K~100K | $0.03~0.10 |
| 코드 리뷰 | 50K~200K | $0.05~0.20 |
| 대규모 리팩토링 | 200K~500K | $0.20~0.50 |
| 프로젝트 분석 | 300K~1M | $0.30~1.00 |

> **주의:** 실제 비용은 프로젝트 크기, 대화 길이, 모델에 따라 달라집니다.

## 6. 팀 예산 관리

### 예산 제한 설정

```bash
# 세션당 최대 비용 제한
claude --max-budget-usd 5.00
```

### 팀 가이드라인 예시

```markdown
## Claude Code 사용 가이드라인

### 모델 선택
- 코드 검색/간단한 질문: Haiku
- 일반 개발 작업: Sonnet (기본)
- 아키텍처/설계 리뷰: Opus (팀 리드 승인)

### 비용 관리
- 일일 예산: 개인 $5, 긴급 시 $10
- /cost로 수시 확인
- 주간 사용량 리포트 공유
```

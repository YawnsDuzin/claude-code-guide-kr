# 성능 및 비용 최적화 가이드

> Claude Code의 응답 속도와 효율성을 높이고, 토큰 비용을 관리하는 실전 가이드입니다.

---

## Part 1: 성능 최적화 전략

### 1. 모델 선택 전략

Claude Code에서는 작업 성격에 따라 모델을 선택할 수 있습니다.

| 모델 | 특징 | 적합한 작업 | 상대 비용 |
|------|------|------------|----------|
| **Opus** | 가장 강력, 복잡한 추론 | 아키텍처 설계, 대규모 리팩토링, 복잡한 버그 | ~10x |
| **Sonnet** | 균형잡힌 성능/속도 | 일반 코딩, 기능 구현, 코드 리뷰 | ~3x |
| **Haiku** | 가장 빠름, 간단한 작업 | 간단한 수정, 포맷팅, 파일 검색 | 1x |

```bash
# 세션 시작 시
claude --model sonnet

# 세션 중 변경
/model sonnet
```

**팁:** 탐색/분석은 Haiku, 구현은 Sonnet, 복잡한 설계는 Opus로 조합하면 비용 대비 효율이 좋습니다.

### 2. 컨텍스트 관리 최적화

컨텍스트가 커질수록 응답이 느려지고 비용이 증가합니다. `/compact`로 대화를 압축하고, 작업 단위로 세션을 분리하세요.

> **상세 가이드:** `/compact` 사용법, 자동 압축 동작, 세션 분리 전략 등은 [컨텍스트 윈도우 관리](../02-핵심개념(core-concepts)/04-컨텍스트관리(context-management).md)를 참조하세요.

### 3. 도구 사용 최적화

#### Glob/Grep 우선 사용

파일 검색에 Bash 대신 전용 도구를 사용하면 더 빠릅니다:

```
# 느림 (Bash)
"find . -name '*.ts' -exec grep 'export class' {} \; 실행해줘"

# 빠름 (전용 도구 유도)
"프로젝트에서 export class가 포함된 TypeScript 파일을 찾아줘"
```

#### 파일 읽기 최적화

```
# 비효율적: 전체 파일 읽기
"src/services/orderService.ts 전체를 읽어줘"

# 효율적: 필요한 부분만
"src/services/orderService.ts의 processOrder 함수만 읽어줘"
```

#### 집중된 프롬프트 작성

```
# 비효율적: 모호하고 넓은 요청
"이 프로젝트를 전체적으로 분석하고 개선점을 모두 알려줘"

# 효율적: 구체적이고 좁은 요청
"src/utils/date.ts의 formatDate 함수에서
타임존 처리 버그를 수정해줘"
```

### 4. 서브에이전트 병렬화

독립적인 작업은 서브에이전트로 병렬 실행하면 빠릅니다:

#### 병렬 처리가 효과적인 경우

```
다음 세 가지를 동시에 분석해줘:
1. src/api/ 디렉토리의 엔드포인트 목록
2. src/models/ 디렉토리의 데이터 모델 구조
3. package.json의 의존성 분석
```

Claude가 자동으로 3개의 서브에이전트를 실행합니다.

#### 병렬 처리가 비효율적인 경우

순차적으로 의존하는 작업:
```
# 이건 순차적으로 진행해야 함
1. 먼저 DB 스키마 설계
2. 스키마 기반으로 API 구현
3. API 기반으로 테스트 작성
```

---

### 5. 배치 작업 최적화

#### 여러 파일 동시 수정

```
# 비효율적: 하나씩 요청
"UserCard.tsx를 수정해줘" → "ProductCard.tsx도 수정해줘" → ...

# 효율적: 한 번에 요청
"components/ 디렉토리의 모든 Card 컴포넌트에서:
- className prop 추가
- 기본 스타일을 Tailwind로 변경
- onClick 핸들러 타입을 MouseEventHandler로 통일"
```

#### 일괄 리팩토링

```
src/api/ 디렉토리의 모든 라우트 핸들러에서:
- try-catch를 asyncHandler 래퍼로 교체
- res.json() 호출을 sendResponse 유틸로 통일
- 에러 응답을 AppError 클래스로 표준화
```

### 6. CLAUDE.md로 반복 줄이기

매 세션마다 반복하는 설명은 CLAUDE.md에 포함시키세요:

```markdown
# 자주 사용하는 컨텍스트

## 에러 핸들링 패턴
모든 서비스는 AppError를 throw하고,
글로벌 에러 핸들러(src/middleware/errorHandler.ts)에서 처리.

## 테스트 패턴
- 파일명: *.test.ts
- 위치: 대상 파일과 같은 디렉토리
- Mock: __mocks__/ 디렉토리
```

이렇게 하면 매번 "우리 프로젝트는 이런 패턴을 써..." 라고 설명할 필요가 없습니다. CLAUDE.md는 캐시 효과로 비용도 줄어듭니다.

## Part 2: 비용 관리

### 8. 비용 구조 이해

#### 토큰이란?

토큰(Token)은 AI 모델이 텍스트를 처리하는 기본 단위입니다. 한국어는 영어보다 더 많은 토큰을 사용합니다.

- 영어: 1단어 ≈ 1~2 토큰
- 한국어: 1글자 ≈ 2~3 토큰
- 코드: 1줄 ≈ 10~30 토큰

#### 입력 vs 출력 토큰

| 구분 | 설명 | 비용 |
|------|------|------|
| **입력 토큰** | 프롬프트, 파일 내용, 컨텍스트 | 상대적으로 저렴 |
| **출력 토큰** | Claude의 응답, 생성된 코드 | 입력 대비 3~5배 비싼 편 |

> **참고:** 정확한 가격은 [Anthropic 가격 페이지](https://www.anthropic.com/pricing)에서 확인하세요.

---

### 9. 사용량 모니터링

#### /cost 명령어

현재 세션의 토큰 사용량을 확인합니다:

```
/cost
```

**출력 예시:**
```
Session cost: $0.42
  Input tokens:  125,000
  Output tokens:  15,000
  Cache reads:    80,000
  Cache writes:   45,000
```

**실시간 모니터링 팁:**
- 대규모 작업 전후에 `/cost` 확인
- 예상보다 비용이 높으면 프롬프트 전략 재검토
- 일일/주간 사용량 추적 습관 만들기

### 10. 사용 계획별 비교

#### Claude Max (구독제)

| 항목 | 내용 |
|------|------|
| 가격 | 월 $100 (Max 5), $200 (Max 20) |
| 특징 | 정액제, 사용량 상한 있음 |
| 장점 | 비용 예측 가능, 초과 과금 없음 |
| 단점 | 한도 초과 시 대기 필요 |
| 적합 | 개인 개발자, 일상적 사용 |

#### API 키 (종량제)

| 항목 | 내용 |
|------|------|
| 가격 | 사용한 토큰만큼 과금 |
| 특징 | 사용량 제한 없음 |
| 장점 | 유연한 사용, 대량 작업 가능 |
| 단점 | 비용 예측 어려움, 관리 필요 |
| 적합 | 팀/기업, 무거운 작업 |

#### 선택 가이드

```
일일 사용 < 2시간 → Max 5 ($100/월)
일일 사용 2~6시간 → Max 20 ($200/월)
일일 사용 > 6시간 또는 CI/CD → API 키
팀 5명 이상 → Team/Enterprise 플랜
```

### 11. 작업별 예상 비용

| 작업 유형 | 예상 토큰 | 대략적 비용 (Sonnet) |
|----------|----------|---------------------|
| 단순 질문/수정 | 5K~15K | < $0.01 |
| 함수 구현 | 15K~50K | $0.01~0.05 |
| 파일 생성 | 30K~100K | $0.03~0.10 |
| 코드 리뷰 | 50K~200K | $0.05~0.20 |
| 대규모 리팩토링 | 200K~500K | $0.20~0.50 |
| 프로젝트 분석 | 300K~1M | $0.30~1.00 |

> **주의:** 실제 비용은 프로젝트 크기, 대화 길이, 모델에 따라 달라집니다.

### 12. 팀 예산 관리

#### 예산 제한 설정

```bash
# 세션당 최대 비용 제한
claude --max-budget-usd 5.00
```

#### 팀 가이드라인 예시

```markdown
## Claude Code 사용 가이드라인

### 모델 선택
- 코드 검색/간단한 질문: Haiku
- 일반 개발 작업: Sonnet (기본)
- 아키텍처/설계 리뷰: Opus (팀 리드 승인)

### 비용 관리
- 일일 예산: 개인 $5, 긴급 시 $10
- /cost로 수시 확인
- 주간 사용량 리포트 공유
```
